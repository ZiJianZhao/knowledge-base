# 2026-05-11 周级扫描：LLM Agent / RL / Memory / Skill

## 值得关注

**[SkillOS: Learning Skill Curation for Self-Evolving Agents](https://arxiv.org/abs/2605.06614)**
一句话：用 RL 训练一个「技能策展者」，让 agent 从经验中自动更新外部 SkillRepo，实现自我进化。
相关性：直接命中 skill learning + agent memory + RL 的交叉点，技能以 Markdown 形式存储并随时间演化为 meta-skill，与你关注的 agent 自我进化方向高度重合。
判断：🔥 建议深潜

**[Belief Memory: Agent Memory Under Partial Observability](https://arxiv.org/abs/2605.05583)**
一句话：将 agent 记忆从"单一确定性结论"改为"多候选结论+概率"，用 Noisy-OR 规则做更新，避免错误自我强化。
相关性：解决 long-term memory 中的核心痛点——agent 在部分可观测环境下的记忆质量，LoCoMo/ALFWorld benchmark 上优于已有 baseline。
判断：🔥 建议深潜

**[RL for LLM-based Multi-Agent Systems through Orchestration Traces](https://arxiv.org/abs/2605.02801)**
一句话：系统梳理 multi-agent RL 的 reward 设计、credit assignment 和编排决策（spawn/delegate/communicate/aggregate/stop）五个子问题，并对齐 Kimi、OpenAI Codex、Claude Code 的工业实践。
相关性：覆盖 multi-agent + tool use + RL 全链路，是当前学术-工业 gap 的一份地图性综述，适合了解领域边界。
判断：👀 值得留意

**[HeavySkill: Heavy Thinking as the Inner Skill in Agentic Harness](https://arxiv.org/abs/2605.02396)**
一句话：把"重度思考"（parallel reasoning → summarization）视为可被 RL 扩展的内化技能，发现它比 Best-of-N 更强，接近 Pass@N 上限。
相关性：skill learning + RL scaling 的视角，论点是"减少对脆弱编排层的依赖，把能力内化进模型参数"，和 SkillOS 形成互补。
判断：👀 值得留意

**[Explaining and Preventing Alignment Collapse in Iterative RLHF](https://arxiv.org/abs/2605.04266)**
一句话：用 Stackelberg 博弈分析 iterative RLHF 中 policy 会系统性利用 RM 盲点（alignment collapse），提出 FPO 通过正则化 policy 对 RM 的参数影响来修复。
相关性：RLHF 训练稳定性的理论分析，如果你在做 reward model 训练或 post-training pipeline，这是值得关注的安全机制。
判断：👀 值得留意

## 本期信号

本周有两个明显趋势：一是 **skill learning 开始与 RL 深度结合**（SkillOS、HeavySkill），agent 不再只是 skill 的使用者，而是用 RL 来学习如何策展和进化 skill；二是 **memory 的不确定性建模**开始被认真对待（BeliefMem），从确定性存储转向概率性存储，是 agent 记忆系统的一个值得关注的新方向。
